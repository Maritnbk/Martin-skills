# 审计报告核对

## 为什么要有这个 Skill

审计报告核对是一项**高频、重复、易出错**的工作。人工核对几十个协会的上百个数据点，难免漏看、算错、忘记。这个 skill 提供了一套系统化的核对流程 + 自动化脚本，帮助您：

- **不漏项** — 三类核对全部覆盖，不会遗漏任何一个检查环节
- **不出错** — 脚本自动做数值比较，避免人工计算失误
- **有记录** — 每个协会的核对结果都写入统一的文件，可追溯、可复查
- **提效率** — 批量处理、并行提取、自动比对，大幅减少手工操作时间

**核心原则：一切以核对结果文件为交付物。** 没有核对结果文件就等于没有做过核对。

## 三类核对概述

详细检查清单和操作方法见 `references/check-details.md`，这里只做概述以便快速理解全局。

### 核对一：报告与汇总表的勾稽关系

**为什么重要：** 报告和汇总表通常来自不同源头（事务所 vs 汇总方），两者不一致意味着至少有一方数据有误。这是最关键的核对。

**核心检查：**
| 检查项 | 说明 |
|--------|------|
| 收入合计 | 报告里写的收入总计 = 汇总表里收入总计列？ |
| 支出合计 | 报告里写的支出总计 = 汇总表里支出总计列？ |
| 收入分类明细 | 政府补助、会费、服务收入、其他收入逐项核对 |
| 支出分类明细 | 业务活动成本、管理费、其他费用逐项核对 |
| 分年度数据 | 汇总表中各年份数据与明细表对应 |
| 收支结余 | 收入 - 支出 = 结余 |
| 专项资金 | 如有赛事经费等专项，单独核对 |

**数值比对技巧：** 使用 `scripts/compare_data.py` 自动比对提取结果中的数值。

### 核对二：报告内部数据勾稽关系

**为什么重要：** 报告自身的数字必须逻辑自洽，否则报告本身质量存疑。

**核心检查：**
- 收入合计 = 各收入分类之和
- 支出合计 = 各支出分类之和  
- 收支结余 = 收入合计 - 支出合计
- 专项资金收支结余逻辑

### 核对三：错别字与语句问题

**为什么重要：** 报告是正式交付文件，文字质量直接影响专业形象。

**高频问题模式（详见 references/common-issues.md）：**
- 缺字："政府拨情况" → "政府拨款情况"
- 错字："认证严格执行" → "认真严格执行"
- 标点：多余句号、英文括号混用
- 格式：数据后缺"元"字
- 表述不通顺：长句缺成分、语序混乱

## 使用自动化脚本

### compare_data.py — 数值自动比对

从提取后的 Markdown 文件中自动提取数值并做三类核对：

```powershell
python scripts/compare_data.py `
  --report "导出目录\协会名_报告.md" `
  --summary "导出目录\协会名_汇总表.md" `
  --output "协会名_核对报告.md"
```

**它能做什么：**
- 自动从 Markdown 中识别报表数值（收入、支出、结余等分类）
- 对比报告与汇总表的对应数值，标注差异
- 验证报告内部勾稽关系（合计 = 分类之和、结余 = 收入 - 支出）
- 输出结构化差异报告

**适用条件：** MinerU 提取后的 Markdown 保留了表格结构。如果提取结果不佳，需要人工辅助核对。

### batch_process.py — 批量处理

```powershell
python scripts/batch_process.py `
  --data-dir "导出目录" `
  --associations "协会A","协会B","协会C" `
  --output "汇总核对结果.md"
```

批量对多个协会运行 compare_data.py 并汇总结果。

## 输出文件格式

核对结果文件 **必须** 包含以下结构（`xxx核对结果.md`）：

```markdown
# 核对结果

## 一、核对范围
列出本次核对的全部协会/组织及文件清单。

---

## 二、各协会核对结果

### （一）[协会名称]

#### 1. 报告与汇总表勾稽关系核对
| 项目 | 报告数据 | 汇总表数据 | 结果 | 备注 |
|------|---------|-----------|:----:|------|

#### 2. 报告内部数据勾稽关系
| 检查项 | 计算结果 | 报告数据 | 结果 |
|--------|---------|---------|:----:|

#### 3. 错别字/语句问题
| 序号 | 位置 | 原文 | 问题描述 |
|:---:|------|------|---------|

---

## 三、问题汇总
按类别统计全部发现。

## 四、重点关注建议
```

> **为什么必须包含"核对范围"？** 明确本次核对了哪些协会、哪些文件，避免后续追溯时产生混淆。
> **为什么"全部通过"也要记？** 核对结果文件本身就是交付物。"全部通过"的记录和"有差异"的记录同样重要。

---

## 关键原则（违反 = 白做）

| 原则 | 说明 |
|------|------|
| **三个核对必须全部执行** | 核对一全部通过 ≠ 核对二、三没问题。跳过任何一个都可能导致漏报。 |
| **有差异要记，无差异也要记** | 核对结果文件是正式交付物，"全部通过"也是有效结论。 |
| **汇总数和明细数都要核对** | 只看合计数可能掩盖分类项中的错误。 |
| **始终参考源文件** | MinerU 提取结果仅供参考，关键数据务必回源文件确认。 |

**停下思考的红色信号：**
- 💡 "数据都对，不用写结果文件了吧" → **必须写**。结果文件是交付物。
- 💡 "只核对总数就行" → **不行**。明细必须逐项核对。
- 💡 "没问题就不用记了" → **要记**。"全部通过"本身就是有效核对结论。

## 常见数值陷阱

| 陷阱 | 说明 | 处理方式 |
|------|------|---------|
| 浮点数精度 | xlsx 中 `5102.910000000001` | 四舍五入到两位小数比较 |
| 公式误差 | xlsx 合计列可能因公式产生微小差异 | 手动重算验证 |
| 千分位逗号 | 报告中 `1,775,477.94` | 去掉逗号后做数值比较 |
| 多层表头 | 非限定性/限定性分列 | 逐列对齐，不要跳列 |
| 多 Sheet | 汇总表可能有收支表+收入明细+支出明细 | 所有 Sheet 都要提取分析 |
| 协会名简称 | 报告中可能用简称 | 统一使用全称做对照 |
