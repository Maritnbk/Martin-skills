# 审计报告核对

## 为什么要有这个 Skill

审计报告核对是一项**高频、重复、易出错**的工作。人工核对审计报告的上百个数据点，难免漏看、算错、忘记。这个 skill 提供了一套系统化的核对流程 + 自动化脚本，帮助您：

- **不漏项** — 三类核对全部覆盖，不会遗漏任何一个检查环节
- **不出错** — 脚本自动做数值比较，避免人工计算失误
- **有记录** — 每个报告的核对结果都写入统一的文件，可追溯、可复查
- **提效率** — 批量处理、并行提取、自动比对，大幅减少手工操作时间

**核心原则：一切以核对结果文件为交付物。** 没有核对结果文件就等于没有做过核对。

## 一、判断核对范围

### 单年度核对
用户只要求核对一个年份的报告（如"核对2023年的报告"）。

### 多年度核对
用户要求核对连续多年度报告（如"核对2023-2025年的报告"或"把三年的报告都核对一下"）。

**核对前必须确认：** 用户到底要核对哪些年份。多年度核对时，**必须逐问确认年份范围**，不要假设。例如用户说"核对这几年的报告"，你应反问"具体是哪几个年份？"

## 二、单年度核对流程（原工作流）

> 当只有一个年份时，按此流程执行。

### 准备阶段
- 确认目录结构（年份文件夹 → 报告.docx + 财务报表.xlsx + 附注.docx）
- 用 MinerU 提取所有文件到临时目录（详细步骤见 `references/extraction-guide.md`）
- 创建核对结果文件

### 对单个年份执行三类核对

详细检查清单见 `references/check-details.md`，这里只做概述。

#### 核对一：报告与财务报表示勾稽关系
核对审计报告正文中的描述性信息与财务报表是否一致（年度、公司名、会计准则、审计意见等）。

#### 核对二：报告内部数据勾稽关系
**为什么重要：** 报告自身的数字必须逻辑自洽。

**A. 财务报表/收支表的内部勾稽**

**资产负债表/财务状况表：**
- 流动资产合计 = 各项流动性资产之和
- 非流动资产合计 = 各项非流动性资产之和
- 资产总计 = 流动资产合计 + 非流动资产合计
- 流动负债合计 = 各项流动负债之和
- 所有者（会员）权益合计 = 实收资本 + 资本公积 + 盈余公积 + 未分配利润
- 负债+所有者权益 = 资产总计

**利润表/业务活动表：**
- 营业利润 = 营业收入 - 营业成本 - 税金及附加 - 销售费用 - 管理费用 - 财务费用
- 利润总额 = 营业利润 + 营业外收入 - 营业外支出
- 净利润 = 利润总额 - 所得税费用

**现金流量表：**
- 经营活动净额 = 各经营现金流入 - 各经营现金流出
- 现金净增加额 = 经营净额 + 投资净额 + 筹资净额
- 期末现金余额 = 期初现金余额 + 现金净增加额

**固定资产勾稽：**
- 固定资产账面价值 = 固定资产原值 - 累计折旧

**B. 附注/明细表的内部勾稽**
- 各账龄段之和 = 对应科目总额（应收/预付/其他应收/应付/预收/其他应付）
- 存货明细（原材料+库存商品+在产品等）= 存货总额
- 固定资产原值/累计折旧/账面价值的各分类明细之和 = 对应合计栏
- 税金及附加的各税种明细之和 = 税金及附加总额
- 各项费用的附注明细之和 = 费用总额
- 实收资本各股东之和 = 实收资本总额

**跨表勾稽关系：**
- 未分配利润（期末-期初）= 净利润 - 提取盈余公积 - 应付利润
- 货币资金 = 现金流量表期末现金余额
- 应交税费附注中各税种明细之和 = 资产负债表应交税费
- 盈余公积（期末-期初）= 提取盈余公积数
- 长期股权投资附注合计 = 资产负债表长期股权投资

**跨年度数据衔接：**
- 本期期初数 = 上期期末数（仅当有上年度数据时）

#### 核对三：错别字与语句问题
逐段阅读报告文本，关注缺字、多字、错字、标点、数字格式、表述不通顺等问题。常见问题模式见 `references/common-issues.md`。

## 三、多年度核对流程（新增）

> 当有多个连续年份时，使用此流程。其核心思想是**两遍扫描**：
> - **第一遍（纵向）**：对每个年份独立执行三类核对，确保各年内部数据自洽
> - **第二遍（横向）**：在所有年份核对完毕后，执行跨年度的横向检查

```
第一遍：纵向核对（每年度独立执行）
┌─────────────────────────────────────────────────┐
│  年份1 → 核对一+核对二+核对三                    │
│  年份2 → 核对一+核对二+核对三                    │
│  年份3 → 核对一+核对二+核对三                    │
│  ...                                             │
│  记录各年度的核对结果到结果文件                    │
└─────────────────────┬───────────────────────────┘
                      ▼
第二遍：横向核对（跨年度检查）
┌─────────────────────────────────────────────────┐
│  (A) 年份完整性检查                              │
│  (B) 跨年度数据衔接检查（逐年对比）               │
│  (C) 跨年度问题传播检查                           │
│  (D) 汇总统计 & 交叉比对                         │
│  记录横向核对结果到结果文件                        │
└─────────────────────────────────────────────────┘
```

---

### 第一遍：纵向核对（Per-Year Vertical Checks）

对**每一个年份**，独立执行单年度流程中的三类核对。

**操作步骤：**
1. 列出所有需要核对的年份
2. 用 MinerU 提取每个年份的 报告.docx、财务报表.xlsx、附注.docx（可并行提取）
3. 从最早的年份开始，逐年完成三类核对
4. 每个年份的核对结果都**写入同一份结果文件的对应年度小节**

**重要：** 每完成一个年份，把该年份的核心数据（特别是期初/期末值）记录下来，为第二遍横向核对做准备。建议用一个结构化表格记录：

```markdown
## 各年份核心数据汇总（仅供横向核对参考）
| 科目 | 2023期末 | 2024期初 | 2024期末 | 2025期初 | 2025期末 |
|------|:-------:|:--------:|:--------:|:--------:|:--------:|
| 货币资金 | 1,513,118.80 | 1,513,118.80 | 2,529,470.89 | 2,529,470.89 | 998,857.18 |
| 应收账款 | ... | ... | ... | ... | ... |
```

这个汇总表**必须**在横向核对前建立，否则无法有效执行跨年度核对。

### 第二遍：横向核对（Cross-Year Horizontal Checks）

#### (A) 年份完整性检查

**目的：** 确保没有任何年份被遗漏。

**操作：**
- 列出用户要求核对的所有年份
- 列出你实际核对的年份
- 两者比对，如果有任何一个年份缺失，标记为"⚠ 年份遗漏"
- 如果有任何一个年份的文件不完整（缺少报告/财务报表/附注中的某项），标记为"⚠ 文件不完整"

```markdown
| 要求核对的年份 | 实际核对 | 报告 | 财务报表 | 附注 | 结果 |
|:-------------:|:--------:|:----:|:--------:|:----:|:----:|
| 2023 | ✅ | ✅ | ✅ | ✅ | ✓ |
| 2024 | ✅ | ✅ | ✅ | ✅ | ✓ |
| 2025 | ✅ | ✅ | ✅ | ✅ | ✓ |
```

**为什么这步重要：** 多年度核对时，很容易因为文件多、年份杂而漏掉某一年。遗漏的年份可能恰恰存在问题，但因为没有核对而被忽略。这步就是那道"安全阀"。

---

#### (B) 跨年度数据衔接检查

**目的：** 确认各科目在年度之间的数据是连续的——上年的期末数应等于下年的期初数。

**操作：**
- 逐一比较：Year N 期末 = Year N+1 期初
- 需要检查的科目：
  - 货币资金、应收账款、预付账款、其他应收款、存货
  - 长期股权投资、固定资产原值、固定资产账面价值
  - 应付账款、预收账款、应交税费、其他应付款
  - 实收资本、盈余公积、未分配利润
  - 资产总计、负债合计、所有者权益合计

```markdown
##### 跨年度数据衔接（2023末 → 2024初）
| 科目 | 2023末 | 2024初 | 结果 |
|------|:-----:|:-----:|:----:|
| 货币资金 | 1,513,118.80 | 1,513,118.80 | ✓ |
```

如有3个以上年度，则每一对相邻年份都需要检查（2023末→2024初, 2024末→2025初, ...）。

---

#### (C) 跨年度问题传播检查 ⭐

**目的：** 找出那些在多个年份中重复出现的问题，以及从一个年份传播到后续年份的问题。

**为什么这步是新增的关键环节：** 单年度核对可能发现某个年份有错，但没有检查这个错误是否也存在于其他年份。实务中最常见的场景是：某个表头错误在第一年出现，后面所有年份都复制了同样的错误；或者某个数据错误导致后续年份的引用数据也错了。

**操作步骤：**

**1. 汇总各年份发现的全部问题（包括核对一、二、三的所有问题）**

| 序号 | 问题描述 | 2023 | 2024 | 2025 | 模式 |
|:----:|---------|:----:|:----:|:----:|------|
| 1 | 应收账款表头误写"占预付账款余额" | ✓ | ✓ | ✓ | **全部年份存在** |
| 2 | 客户名称"陝西"使用繁体字 | ✓ | ✓ | ✗ | **部分年份存在，后续已修正** |
| 3 | 未分配利润表净利润与利润表不一致 | ✓ | ✗ | ✗ | **仅首次出现** |

**2. 按传播模式分类：**

| 模式 | 含义 | 处理方式 |
|------|------|---------|
| **全部年份存在** | 所有年份都存在同样的问题 | 建议统一修正 |
| **首次出现后传播** | 问题在Year N出现，并影响Year N+1的数据引用 | 需要联动修正 |
| **部分年份存在，后续已修正** | 早期年份有错，后期已修正 | 建议回溯修正早期年份 |
| **仅单年度存在** | 只出现在一个年份，与其他年份无关 | 单独修正即可 |
| **年份间模式不同** | 问题在不同年份的表现形式不同 | 分别记录，独立修正 |

**3. 特别关注"引用传播"问题：**

某些年份的数据错误会"传染"到后续年份，因为后续年份的报告会引用上期数据。典型场景：
- Year N 的未分配利润表出错了 → Year N+1 引用"上期金额"时也跟着错
- Year N 的某个数值录错了 → Year N+1 的期初余额引用时也错了
- 某个科目的名称在所有年份中都错了（模板复制导致）

检查方法：查看后续年份附注中的"上期金额""期初余额"是否与之前年份的"本期金额""期末余额"一致。如果不一致，再判断是引用错误还是Year N本身错误。

#### (D) 汇总统计与交叉比对

**目的：** 生成完整的跨年度汇总统计，确保结论覆盖所有年份。

**操作：**

**1. 按类别统计问题**

```markdown
| 类别 | 2023 | 2024 | 2025 | 合计 |
|------|:----:|:----:|:----:|:----:|
| 数据异常 | 2 | 0 | 0 | 2 |
| 繁体字/错字 | 1 | 1 | 0 | 2 |
| 表头文字错误 | 1 | 1 | 1 | 3 |
| 文号年份错误 | 1 | 0 | 0 | 1 |
| **合计** | **5** | **2** | **1** | **8** |
```

**2. 总计校验结果**

```markdown
| 分类 | 项目数 | 通过 | 异常 | 通过率 |
|------|:-----:|:----:|:---:|:-----:|
| 2023年数值勾稽 | 38 | 36 | 2 | 94.7% |
| 2024年数值勾稽 | 38 | 38 | 0 | 100.0% |
| 2025年数值勾稽 | 39 | 39 | 0 | 100.0% |
| 跨年度数据衔接 | 38 | 38 | 0 | 100.0% |
| **合计** | **153** | **151** | **2** | **98.7%** |
```

**3. 详细问题清单（按全部分年度排序）**

| 优先级 | 年度 | 问题描述 | 建议处理 |
|:------:|:----:|---------|---------|
| **高** | 2023 | 未分配利润表净利润与利润表不一致 | 核实修正 |
| **中** | 2023-2025 | 应收账款表头错误（三年均存在） | 统一修正 |
| **低** | 2024-2025 | 数字格式错误 | 修正格式 |

### 多年度核对的核心操作顺序

为避免遗漏，请严格按照以下顺序操作：

```
Step 1: 列出所有年份，确认文件齐全
Step 2: 提取所有年份的文件（并行）
Step 3: 从最早年份开始，逐年执行三类核对 → 写入结果文件
Step 4: 建立各年份核心数据汇总表
Step 5: 执行横向核对：
  5a. 年份完整性检查
  5b. 跨年度数据衔接检查
  5c. 跨年度问题传播检查
  5d. 汇总统计与交叉比对
Step 6: 生成最终结果文件
```

## 多年度核对的关键原则

| 原则 | 说明 |
|------|------|
| **先纵向再横向** | 必须对所有年份先独立完成三类核对，再做横向跨年度检查。跳过纵向直接做横向会导致遗漏。 |
| **年份完整性必须检查** | 多年度核对的第一步就是确认所有年份都在。遗漏年份是最容易犯的错误。 |
| **建立核心数据汇总表** | 横向核对前必须汇总各年份的关键数据到一个表格，否则无法有效比较。 |
| **问题传播必须检查** | 单个年份发现的问题可能也在其他年份存在，必须显式检查。特别是"引用传播"——前一年错了后一年跟着错。 |
| **汇总统计覆盖全部年份** | 总计校验结果的分子分母必须包含所有年份和所有横向检查项。 |
| **每完成一个年份就记录** | 不要等全部做完才写结果文件，逐年份写入可以避免遗忘和混淆。 |

**停下思考的红色信号（多年度特有）：**
- 💡 "先做完今年的核对，去年的后面再说" → **不行**，多年度核对必须系统性地逐年份执行
- 💡 "三年数据看起来都差不多，应该没问题" → **必须逐项核对**，不能靠感觉
- 💡 "这个错误在2023年发现了，2024/2025应该也有" → **必须显式检查确认**，不能假设
- 💡 "跨年度的问题汇总等最后再说" → **现在就开始记录**，每发现一个年份的问题就标记是否可能跨年

## 常见数值陷阱

| 陷阱 | 说明 | 处理方式 |
|------|------|---------|
| 浮点数精度 | xlsx 中 `5102.910000000001` | 四舍五入到两位小数比较 |
| 公式误差 | xlsx 合计列可能因公式产生微小差异 | 手动重算验证 |
| 千分位逗号 | 报告中 `1,775,477.94` | 去掉逗号后做数值比较 |
| 多层表头 | 非限定性/限定性分列 | 逐列对齐，不要跳列 |
| 多 Sheet | 汇总表可能有收支表+收入明细+支出明细 | 所有 Sheet 都要提取分析 |
| 协会名简称 | 报告中可能用简称 | 统一使用全称做对照 |
