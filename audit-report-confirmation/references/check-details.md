# 三类核对详细检查清单

本文件在 SKILL.md 概述基础上提供逐项操作指南。当您在执行某一类核对时需要参考具体步骤时，阅读对应章节。

> **设计原则：** 这些清单不是因为"流程需要"而存在，而是因为审计核对实践中每个环节都出过真实问题。每一条背后都有实际案例支撑。

---

## 核对一：报告与汇总表勾稽关系

### 目的

确认审计报告(docx)中的财务数据与汇总表(xlsx)的数据一致。这是三类核对中最核心的环节——两个独立来源的数据如果不一致，说明至少有一方有误。

### 操作步骤

#### Step 1：定位数据位置

分别在报告提取结果和汇总表提取结果中找到财务数据：

- **报告侧：** 通常在"财务状况"或"收支情况"章节，以报表形式呈现
- **汇总表侧：** 可能有多个 Sheet（收支汇总表、收入明细表、支出明细表等），需要全部分析

#### Step 2：核对汇总数（宏观确认）

| 检查项 | 报告值 | 汇总表值 | 是否一致 |
|--------|--------|----------|:--------:|
| 收入合计 | | | |
| 支出合计 | | | |
| 收支结余 | | | |

如果汇总数不一致，重点排查：是否漏看了某一行、是否包含了/排除了专项资金。

#### Step 3：核对明细分类

**收入明细：**
| 分类 | 报告值 | 汇总表值 | 是否一致 |
|------|--------|----------|:--------:|
| 政府补助收入 | | | |
| 会费收入 | | | |
| 提供服务收入 | | | |
| 其他收入 | | | |

**支出明细：**
| 分类 | 报告值 | 汇总表值 | 是否一致 |
|------|--------|----------|:--------:|
| 业务活动成本 | | | |
| 管理费用 | | | |
| 其他费用 | | | |

#### Step 4：核对分年度数据（如果适用）

如果汇总表按年份列示数据，需要逐条核对各年份的数值：

| 年份 | 报告值 | 汇总表值 | 是否一致 |
|:----:|--------|----------|:--------:|

#### Step 5：核对专项资金（如果存在）

如报告中有"赛事经费""专项资金""项目经费"等特殊章节，单独核对其收入、支出、结余。

### 数值处理规则

| 源格式 | 处理方法 | 示例 |
|--------|---------|------|
| `1,775,477.94`（千分位逗号） | 去掉逗号 | → `1775477.94` |
| `5102.910000000001`（浮点数精度） | 四舍五入到2位小数 | → `5102.91` |
| `5,102.91`（含千分位+小数） | 去掉逗号再比较 | → `5102.91` |
| 公式计算结果 | 手动重算验证 | — |

### 异常处理

| 异常 | 可能原因 | 处理方式 |
|------|---------|---------|
| 相差 0.01 | 四舍五入方式不同 | 记录为"精度差异" |
| 相差较大 | 数据源不同/计算口径不同 | 标记为"差异"，需人工确认 |
| 汇总表缺少某个明细项 | 该协会可能无此项收入/支出 | 报告中确认为0即可 |

## 核对二：报告内部数据勾稽关系

### 目的

确认报告自身数据在逻辑上自洽。即使报告与汇总表一致，报告本身的内部逻辑可能有问题（如分项之和≠合计）。

### 操作步骤

#### Step 1：收入勾稽

```text
收入合计 = 政府补助收入 + 会费收入 + 提供服务收入 + 其他收入
```

逐项取出数值，做加法验证。

#### Step 2：支出勾稽

```text
支出合计 = 业务活动成本 + 管理费用 + 其他费用
```

#### Step 3：收支结余勾稽

```text
收支结余 = 收入合计 - 支出合计
```

注意：结余可能是负数（支大于收），这是正常的。

#### Step 4：专项资金内部勾稽

```text
专项资金结余 = 专项资金收入 - 专项资金支出
```

### 常见失败模式

| 模式 | 示例 | 原因 |
|------|------|------|
| 合计 ≠ 分项之和 | 收入合计 100万，四项收入之和 98万 | 漏列了某项收入分类 |
| 结余计算错误 | 收支结余与收入-支出不一致 | 可能是人为录入错误 |
| 专项与常规数据交叉 | 专项资金数据被重复计入常规收支 | 区分不清晰导致 |

## 核对三：错别字与语句问题

### 目的

审计报告是正式专业文件，文字质量直接影响协会和审计方的专业形象。

### 操作步骤

#### Step 1：全文通读

逐段阅读报告文本，关注：

| 关注点 | 常见表现 |
|--------|---------|
| 缺字 | "政府拨情况"（缺"款"）、"未政府补助收入"（缺"收到"） |
| 多字 | "报告期内期内"、多余空格 |
| 错字 | "认证严格执行"（应为"认真"）、"关与"（应为"关于"）、"令一方面"（应为"另一方面"）|
| 标点 | 多余句号"。。"、英文括号混用")"、中文引号不配对 |
| 数字格式 | "530,831.78"后缺"元"、"2024年度"应为"2024年" |
| 语序/成分残缺 | 长句缺少主语、动宾搭配不当 |

#### Step 2：重点段落复核

以下段落是错别字高频区：

- 报告开头和结尾（套话模板容易出问题）
- 财务数据段落（数字和文字混合处）
- 新增或修改的段落（新写内容 > 复制粘贴内容）

#### Step 3：统一性检查

| 检查项 | 示例 |
|--------|------|
| 协会名称在全文中是否统一 | 不可一会用全称，一会用简称 |
| 年度表述是否一致 | "2024年度" vs "2024年" |
| 货币单位是否标注 | 数据后是否都有"元" |
| 表格格式是否一致 | 标题行对齐方式、字号等 |
