## 核对一：报告与汇总表勾稽关系

### 目的

确认审计报告(docx)中的财务数据与汇总表(xlsx)的数据一致。这是三类核对中最核心的环节——两个独立来源的数据如果不一致，说明至少有一方有误。

### 操作步骤

#### Step 1：定位数据位置

分别在报告提取结果和汇总表提取结果中找到财务数据：

- **报告侧：** 通常在"财务状况"或"收支情况"章节，以报表形式呈现
- **汇总表侧：** 可能有多个 Sheet（收支汇总表、收入明细表、支出明细表等），需要全部分析

#### Step 2：核对汇总数（宏观确认）

| 检查项 | 报告值 | 汇总表值 | 是否一致 |
|--------|--------|----------|:--------:|
| 收入合计 | | | |
| 支出合计 | | | |
| 收支结余 | | | |

如果汇总数不一致，重点排查：是否漏看了某一行、是否包含了/排除了专项资金。

#### Step 3：核对明细分类

**收入明细：**
| 分类 | 报告值 | 汇总表值 | 是否一致 |
|------|--------|----------|:--------:|
| 政府补助收入 | | | |
| 会费收入 | | | |
| 提供服务收入 | | | |
| 其他收入 | | | |

**支出明细：**
| 分类 | 报告值 | 汇总表值 | 是否一致 |
|------|--------|----------|:--------:|
| 业务活动成本 | | | |
| 管理费用 | | | |
| 其他费用 | | | |

#### Step 4：核对分年度数据（如果适用）

如果汇总表按年份列示数据，需要逐条核对各年份的数值：

| 年份 | 报告值 | 汇总表值 | 是否一致 |
|:----:|--------|----------|:--------:|

#### Step 5：核对专项资金（如果存在）

如报告中有"赛事经费""专项资金""项目经费"等特殊章节，单独核对其收入、支出、结余。

### 数值处理规则

| 源格式 | 处理方法 | 示例 |
|--------|---------|------|
| `1,775,477.94`（千分位逗号） | 去掉逗号 | → `1775477.94` |
| `5102.910000000001`（浮点数精度） | 四舍五入到2位小数 | → `5102.91` |
| `5,102.91`（含千分位+小数） | 去掉逗号再比较 | → `5102.91` |
| 公式计算结果 | 手动重算验证 | — |

### 异常处理

| 异常 | 可能原因 | 处理方式 |
|------|---------|---------|
| 相差 0.01 | 四舍五入方式不同 | 记录为"精度差异" |
| 相差较大 | 数据源不同/计算口径不同 | 标记为"差异"，需人工确认 |
| 汇总表缺少某个明细项 | 该协会可能无此项收入/支出 | 报告中确认为0即可 |

## 核对二：报告内部数据勾稽关系

### 目的

确认报告自身数据在逻辑上自洽。即使报告与汇总表一致，报告本身的内部逻辑也可能有问题（如分项之和≠合计、跨表数据不一致、附注明细加总错误、期初期末不衔接）。

### 核对原则

核对二涉及多个层面，实务中常达数十到上百个检查项。建议按以下顺序逐层检查，**每通过一层就记录一层，无差异也要记**：

```
报表层面 → 附注层面 → 跨表勾稽 → 跨年度衔接
```

### 第一部分：报表层面的内部勾稽

#### Step 1A：资产负债表逐项加总验证

**为什么：** 资产负债表的合计数往往由公式自动计算，但 xlsx 公式可能出错、人为复制粘贴可能覆盖公式、跨 Sheet 引用可能断开。

逐项取出各分类的明细金额，做加法验证：

**流动资产合计验证：**
```text
流动资产合计 = 货币资金 + 短期投资 + 应收票据 + 应收账款 + 预付账款 
               + 应收股利 + 应收利息 + 其他应收款 + 存货 + 其他流动资产
```
- 如果某科目值为空或0，加总时按0处理
- 验证结果是否与报表中的"流动资产合计"一致

**非流动资产合计验证：**
```text
非流动资产合计 = 长期股权投资 + 固定资产账面价值 + 在建工程 + 工程物资 
                + 无形资产 + 长期待摊费用 + 其他非流动资产
```

**资产总计验证：**
```text
资产总计 = 流动资产合计 + 非流动资产合计
```

**流动负债合计验证：**
```text
流动负债合计 = 短期借款 + 应付票据 + 应付账款 + 预收账款 + 应付职工薪酬 
              + 应交税费 + 应付利息 + 应付利润 + 其他应付款 + 其他流动负债
```

**所有者权益合计验证：**
```text
所有者（或会员）权益合计 = 实收资本 + 资本公积 + 盈余公积 + 未分配利润
```

**终极平衡验证：**
```text
负债 + 所有者权益 = 资产总计
```

#### Step 1B：利润表分层验证

**为什么：** 利润表不是单纯的一层"收入-支出=结余"，而是多层递进结构。每一层都可能出错。

**营业利润验证：**
```text
营业利润 = 营业收入 - 营业成本 - 税金及附加 - 销售费用 - 管理费用 - 财务费用
```
注意财务费用可能是负数（利息收入 > 利息支出），减负数等于加正数。

**利润总额验证：**
```text
利润总额 = 营业利润 + 营业外收入 - 营业外支出
```

**净利润验证：**
```text
净利润 = 利润总额 - 所得税费用
```

#### Step 1C：现金流量表分层验证

**为什么：** 现金流量表的勾稽关系比利润表更易出错，因为明细行项多，且各活动的加总横跨三大类别。

**各活动净额验证：**
```text
经营活动净额 = 经营活动现金流入小计 - 经营活动现金流出小计
投资活动净额 = 投资活动现金流入小计 - 投资活动现金流出小计
筹资活动净额 = 筹资活动现金流入小计 - 筹资活动现金流出小计
```

**现金净增加额验证：**
```text
现金净增加额 = 经营活动净额 + 投资活动净额 + 筹资活动净额
```

**期末现金余额验证：**
```text
期末现金余额 = 期初现金余额 + 现金净增加额
```
此数应等于资产负债表的"货币资金"期末数（跨表勾稽，见 Step 3）。

#### Step 1D：固定资产勾稽验证

**为什么：** 固定资产原值-累计折旧=账面价值是最基本的会计恒等式，但实务中常因折旧计算错误或原值变动未同步更新而出现不等。

```text
固定资产账面价值 = 固定资产原值 - 累计折旧
```
如有分类明细（机器设备、运输设备、电子设备等），分别验证每类的原值-折旧=账面价值，再验证明细之和=合计。

固定资产变动表（如有）的验证：
```text
期末原值 = 期初原值 + 本期增加 - 本期减少
期末累计折旧 = 期初累计折旧 + 本期计提折旧 - 本期减少转出
期末账面价值 = 期末原值 - 期末累计折旧
```

### 第二部分：附注/明细表的内部勾稽

#### Step 2A：附注定性的理解

附注是对报表数据的具体分解说明。同一笔合计数在附注中被拆分为多个明细行，必须验证**明细之和 = 合计数**。这是最容易出错也最容易被忽略的校验。

#### Step 2B：账龄类科目的明细加总

**应收/预付/其他应收款、应付/预收/其他应付款**等科目通常按账龄（1年以内、1-2年、2-3年、3年以上）列示：

```text
合计 = 1年以内金额 + 1-2年金额 + 2-3年金额 + 3年以上金额
```
- 同时也验证各账龄段的百分比之和是否等于100%
- 如有前5大客户明细，验证其合计金额与占比

#### Step 2C：存货明细加总

```text
存货合计 = 原材料 + 库存商品/产成品 + 在产品 + 周转材料 + ...
```
分别验证期末和期初的明细之和。

#### Step 2D：固定资产各分类明细加总

固定资产表通常按类别列示：

```text
原值合计 = 机器设备原值 + 运输设备原值 + 电子设备原值 + ...
累计折旧合计 = 机器设备折旧 + 运输设备折旧 + 电子设备折旧 + ...
账面价值合计 = 机器设备净值 + 运输设备净值 + 电子设备净值 + ...
```

**特别重要：不仅要验证期末数，还要验证期初数。** 实务中曾发现合计栏与明细之和相差10元的案例，仅出现在期初数。

#### Step 2E：税金及附加明细加总

```text
税金及附加合计 = 城建税 + 教育费附加 + 地方教育附加 + 印花税 + 城镇土地使用税 + ...
```

#### Step 2F：各项费用的附注明细加总

**销售费用、管理费用、财务费用**等通常在附注中列出各明细项目（差旅费、广告费、服务费、职工薪酬、折旧费等）：

```text
销售费用总额 = 差旅费 + 广告费 + 运输费 + 业务招待费 + 办公费 + ...
管理费用总额 = 研究费用 + 职工薪酬 + 服务费 + 社保费 + 房租费 + 折旧费 + ...
财务费用总额 = 利息净支出 + 银行手续费 + ...
```

**验证两边：** 明细之和 = 附注中该费用总额 = 利润表中对应费用行

#### Step 2G：实收资本明细加总

```text
实收资本合计 = 各股东出资额之和
```
验证股东出资比例之和 = 100%

#### Step 2H：关联方应收应付款验证

可结合关联方关系表，验证：

```text
关联方应收合计 = 各关联方应收明细之和
关联方应付合计 = 各关联方应付明细之和
```
同时可以交叉验证关联方应收/应付是否与账龄表中相关明细一致（如预付某关联方的金额与预付账款明细中该客户金额一致）。

#### Step 2I：未分配利润表内部逻辑验证

```text
可供分配利润 = 净利润 + 年初未分配利润
未分配利润 = 可供分配利润 - 提取法定盈余公积 - 提取任意盈余公积 - 应付利润
```
验证期末未分配利润与资产负债表一致。

### 第三步：跨表勾稽关系

#### 为什么重要

不同的财务报表（资产负债表、利润表、现金流量表、附注）之间存在逻辑关联。这些跨表关系经常被忽视，但一旦出现不一致，说明至少有一张表的数据有误。

#### Step 3A：未分配利润变动勾稽

这是最重要的跨表勾稽之一：
```text
未分配利润（期末 - 期初） = 净利润 - 提取盈余公积 - 应付利润
```
两边分别计算，验证相等。

#### Step 3B：盈余公积变动勾稽

```text
盈余公积（期末 - 期初） = 当期提取盈余公积数
```
当某年度提取了盈余公积时此勾稽关系生效；未提取时盈余公积期末=期初。

#### Step 3C：货币资金与现金流量表勾稽

```text
资产负债表"货币资金"期末数 = 现金流量表"期末现金余额"
```
这是最基本、也最重要的跨表勾稽。

#### Step 3D：应交税费与附注勾稽

```text
资产负债表"应交税费" = 附注中各税种明细（增值税+城建税+教育费附加+地方教育附加+所得税+印花税+...）之和
```

#### Step 3E：附注中费用合计与利润表勾稽

已在 Step 2F 中覆盖——附注中销售费用/管理费用/财务费用/税金及附加的合计应等于利润表对应行。

### 第四步：跨年度数据衔接

#### 为什么重要

跨年度衔接错误可能意味着"今年的期初数 =/= 去年的期末数"，这通常是因为去年审计调整未入账、数据未过账或人为录入错误。这种错误一旦发生，会影响连续两年的报表。

#### 操作步骤

逐一核对以下科目的跨年衔接：

| 科目 | 202X末 → 202X+1初 | 结果 |
|------|:-----------------:|:----:|
| 货币资金 | | |
| 应收账款 | | |
| 预付账款 | | |
| 其他应收款 | | |
| 存货 | | |
| 长期股权投资 | | |
| 固定资产原值 | | |
| 固定资产账面价值 | | |
| 应付账款 | | |
| 预收账款 | | |
| 应交税费 | | |
| 实收资本 | | |
| 未分配利润 | | |
| 资产总计 | | |

如果一个批次的年度较多（如3年），则需检查每一年的衔接：2023末→2024初→2025初→...

---

### 常见失败模式

| 模式 | 示例 | 原因 |
|------|------|------|
| 合计 ≠ 分项之和 | 收入合计100万，四项收入之和98万 | 漏列了某项收入分类或人为录入错误 |
| 结余计算错误 | 收支结余与收入-支出不一致 | 人为录入错误 |
| 专项与常规数据交叉 | 专项资金数据被重复计入常规收支 | 区分不清晰导致 |
| **资产负债表不平衡** | 资产≠负债+所有者权益 | 公式错误、漏行、复制粘贴覆盖了公式 |
| **利润表分层错误** | 营业利润计算遗漏了某项费用 | 多层计算中某一层出错 |
| **附注明细合计≠合计数** | 账龄4个段之和≠合计栏 | 录入时漏了一项或合计数未更新 |
| **固定资产原值-折旧≠账面价值** | 折旧计算有误或原值变更未同步 | 折旧计提或资产处置未正确记录 |
| **跨表不一致** | 货币资金 ≠ CF期末现金 | 数据来源不同步 |
| **跨年不衔接** | 某科目今年期初≠去年期末 | 审计调整未入账 |
| **期初附注明细与合计不一致** | 期初账面价值明细之和≠合计栏，但期末一致 | 期初数据录入时出错的典型特征 |

## 核对三：错别字与语句问题

### 目的

审计报告是正式专业文件，文字质量直接影响协会和审计方的专业形象。

### 操作步骤

#### Step 1：全文通读

逐段阅读报告文本，关注：

| 关注点 | 常见表现 |
|--------|---------|
| 缺字 | "政府拨情况"（缺"款"）、"未政府补助收入"（缺"收到"） |
| 多字 | "报告期内期内"、多余空格 |
| 错字 | "认证严格执行"（应为"认真"）、"关与"（应为"关于"）、"令一方面"（应为"另一方面"）|
| 标点 | 多余句号"。。"、英文括号混用")"、中文引号不配对 |
| 数字格式 | "530,831.78"后缺"元"、"2024年度"应为"2024年" |
| 语序/成分残缺 | 长句缺少主语、动宾搭配不当 |

#### Step 2：重点段落复核

以下段落是错别字高频区：

- 报告开头和结尾（套话模板容易出问题）
- 财务数据段落（数字和文字混合处）
- 新增或修改的段落（新写内容 > 复制粘贴内容）

#### Step 3：统一性检查

| 检查项 | 示例 |
|--------|------|
| 协会名称在全文中是否统一 | 不可一会用全称，一会用简称 |
| 年度表述是否一致 | "2024年度" vs "2024年" |
| 货币单位是否标注 | 数据后是否都有"元" |
| 表格格式是否一致 | 标题行对齐方式、字号等 |

## 多年度横向核对：年份完整性检查

### 目的

确认所有应该核对的年份都被实际核对到了。多年度核对中，遗漏年份是最高频的错误之一。

### 操作步骤

1. **收集要求：** 从用户处确认需要核对的年份列表
2. **扫描文件：** 检查每个年份目录是否包含以下文件：
   - 报告正文（如 `3-审计报告正文.docx`）
   - 财务报表（如 `4-财务报表模板.xlsx`）
   - 附注（如 `5-附注模板.docx`）
3. **建立核对表：**

```markdown
| 要求核对的年份 | 报告正文 | 财务报表 | 附注 | 核对状态 |
|:-------------:|:--------:|:--------:|:----:|:--------:|
| 2023 | ✅ | ✅ | ✅ | ✅ 已完成 |
| 2024 | ✅ | ✅ | ✅ | ✅ 已完成 |
| 2025 | ✅ | ❌ 缺失 | ✅ | ❌ 跳过 |
```

4. **处理缺失：**
   - 如果某年份文件齐全但未核对 → 补充核对
   - 如果某年份文件缺失 → 在结果文件中标记为"文件不完整，跳过"
   - 如果某年份用户要求了但目录中不存在 → 告知用户，请用户确认

## 多年度横向核对：跨年度数据衔接

### 目的

确认相邻年份之间的数据是连续的——上年期末数 = 下年期初数。这是多年度核对的核心检查项。

### 操作步骤

1. **建立核心数据汇总表：** 从各年份的资产负债表中提取关键科目的期初/期末数据，放入一个汇总表。这是横向核对的基础。

```markdown
### 各年份核心数据汇总
| 科目 | 2022末(2023初) | 2023末(2024初) | 2024末(2025初) |
|------|:-------------:|:-------------:|:-------------:|
| 货币资金 | 1,095,402.25 | 1,513,118.80 | 2,529,470.89 |
| 应收账款 | 7,563,619.73 | 6,808,109.26 | 12,763,843.13 |
| ... | ... | ... | ... |
```

2. **逐年比较：** 对每对相邻年份，逐一比较各科目
   - Year N 期末 = Year N+1 期初？→ ✓
   - Year N 期末 ≠ Year N+1 期初？→ ⚠ 记录差异

3. **需要检查的科目列表：**

| 资产类 | 负债类 | 权益类 |
|--------|--------|--------|
| 货币资金 | 应付账款 | 实收资本 |
| 应收账款 | 预收账款 | 盈余公积 |
| 预付账款 | 应交税费 | 未分配利润 |
| 其他应收款 | 其他应付款 | 所有者权益合计 |
| 存货 | 负债合计 | |
| 长期股权投资 | | |
| 固定资产原值 | | |
| 固定资产账面价值 | | |
| 资产总计 | | |

4. **识别异常模式：**
   - 大多数科目正常仅个别科目异常 → 可能为录入错误
   - 所有科目都有微小差异（如差0.01） → 可能是四舍五入差异
   - 所有科目都一致但资产总计不一致 → 可能漏行或公式错误

## 多年度横向核对：跨年度问题传播检查 ⭐

### 目的

找出那些在多个年份中重复出现或者从一个年份传播到另一个年份的问题。这是多年度核对区别于单年度核对的核心所在。

### 为什么需要专门的传播检查

单年度核对只能发现"这个年份有这个错误"。但很多错误不是孤立存在的：
- 表头错误会在所有年份中重复出现（模板复制导致）
- 一个年份的数值错误会影响后续年份的引用
- 某些问题在早期年份存在，后期已修正
- 某些问题在后期年份才出现，但原因在早期年份

如果不做传播检查，这些跨年份的关联就会被忽略。

### 操作步骤

#### Step 1：汇总各年份的全部发现

把核对一、核对二、核对三在每个年份中发现的所有问题汇总到一张表中：

```markdown
| # | 问题描述 | 2023 | 2024 | 2025 |
|:-:|---------|:----:|:----:|:----:|
| 1 | 应收账款表头误写"占预付账款余额的比例" | ✓ 存在 | ✓ 存在 | ✓ 存在 |
| 2 | 客户名称"陝西"使用繁体字 | ✓ 存在 | ✓ 存在 | ✗ 已修正 |
| 3 | 未分配利润表净利润与利润表不一致 | ✓ 存在 | ✗ 无 | ✗ 无 |
| 4 | 数字格式错误"81,72,962.25" | ✗ 无 | ✓ 存在 | ✓ 存在 |
```

#### Step 2：分析传播模式

对于每个问题，判断其跨年度传播模式：

| 传播模式 | 判断方法 | 处理建议 |
|---------|---------|---------|
| **全部年份存在** | 所有年份都有此问题 | 统一修正，检查模板源文件 |
| **首次出现，后续传播** | 在 Year N 首次出现，Year N+1 及以后也有 | 联动修正，注意引用关系 |
| **早期存在，后期已修正** | 早期年份有，最新年份已无 | 建议回溯修正早期年份，保持一致性 |
| **仅单年度存在** | 只有一个年份有 | 单独修正即可，无需联动 |
| **后期才出现** | 早期年份无，后期年份有 | 检查变化原因，单独修正 |

#### Step 3：检查引用传播（最易遗漏）

某些问题会通过"引用关系"从一个年份传播到另一个年份：
- 附注中"上期金额"引用上年"本期金额"
- 资产负债表的"期初余额"引用上年"期末余额"
- 未分配利润表中的"年初未分配利润"

**检查方法：**

```
如果 Year N 附注的"未分配利润表"中的净利润有误，
那么 Year N+1 附注中引用的"上期金额"也可能有误。
→ 需要检查 Year N+1 的附注是否直接复制了 Year N 的错误数据
```

**典型引用传播场景：**

| 场景 | 示例 |
|------|------|
| 附注未分配利润表净利润错误 → 下年上期金额引用 | 2023年净利润51,705.42（错），2024年引用"上期净利润51,705.42"也错（如果直接复制） |
| 固定资产账面价值错误 → 下年期初余额 | 附注固定资产期初数据错误 → 下年附注引用时也可能跟着错 |
| 表头文字错误 → 多年模板复制 | 应收账款表头误写为"预付账款"，三年报告都错 |

#### Step 4：汇总传播检查结果

```markdown
### 跨年度问题传播检查结果
| # | 问题 | 传播模式 | 涉及年度 | 建议处理 |
|:-:|------|---------|:--------:|---------|
| 1 | 应收账款表头"占预付账款..." | 全部年份存在 | 2023-2025 | 统一修正模板 |
| 2 | 客户名称繁体字"陝西" | 早期存在，后期修正 | 2023-2024 | 回溯修正早期年份 |
| 3 | 未分配利润净利润51,705.42 | 首次出现（但被下年引用） | 2023-2025 | 确认修正后检查后续引用 |
```
